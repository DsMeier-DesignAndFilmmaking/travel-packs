# Local City Travel Packs

Production-ready scaffold for a global, offline-first travel platform built with Vite + React + TypeScript.

## Architecture goals
- UI is fully data-driven from external JSON files.
- Content model is decoupled from rendering components.
- Repository layer allows migration from static files to APIs/CDN.
- Routing is slug-based for large-scale city pack catalogs.
- PWA plumbing is configured but disabled until caching strategy is defined.

## Scalability notes
- **Thousands of packs**: each pack is isolated in a dedicated JSON file addressed by slug.
- **JSON-driven content**: views consume typed entities from `types/` and `services/` only.
- **AI-generated data ready**: schema supports extensible sections/content blocks.
- **Future API integrations**: swap `contentClient` internals without touching feature UI.
